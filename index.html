🚀 启用云端多设备同步

完成以下3步配置，让员工可通过链接实时填报

1

注册 Supabase 免费账号（免费，无需信用卡）

Supabase 是免费的云数据库服务，国内可直接访问。

  1. 打开 supabase.com → 点击 Start your project → 用 GitHub 登录
  2. 点击 New Project → 填写项目名（如 rework）→ 设置数据库密码 → Create
  3. 等待约1分钟项目启动完成
  4. 进入项目 → 左侧 Settings → API → 复制 Project URL 和 anon public Key
2

在 Supabase 创建数据表

在 Supabase 控制台 → SQL Editor → 粘贴并运行以下SQL

CREATE TABLE IF NOT EXISTS rework_records (
  id TEXT PRIMARY KEY,
  date TEXT NOT NULL,
  dept TEXT NOT NULL,
  product_name TEXT NOT NULL,
  batch_no TEXT,
  qty INTEGER NOT NULL DEFAULT 1,
  operator TEXT NOT NULL,
  defect_type TEXT NOT NULL,
  reason TEXT,
  dispose TEXT NOT NULL,
  work_hours NUMERIC DEFAULT 0,
  remark TEXT,
  photos JSONB DEFAULT '[]'::jsonb,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ
);
ALTER TABLE rework_records ENABLE ROW LEVEL SECURITY;
CREATE POLICY IF NOT EXISTS "allow_all" ON rework_records
  FOR ALL USING (true) WITH CHECK (true);
3

将本页面发布到公网（获取可分享链接）

选择一种方式，将此 HTML 文件托管到公网：

⭐ 方式A：GitHub Pages（推荐新手） ▼

1. 注册 github.com → New repository → 设为 Public → Create

2. 点击 uploading an existing file → 将本 HTML 文件名改为 index.html → 上传 → Commit

3. Settings → Pages → Source 选 main 分支 → Save

4. 几分钟后获得：https://用户名.github.io/仓库名/

5. 员工填报链接：https://用户名.github.io/仓库名/?mode=submit

🚀 方式B：Cloudflare Pages（国内访问更快） ▼

1. 注册 Cloudflare → Workers & Pages → Pages

2. Create application → Upload assets → 上传本 HTML（改名为 index.html）

3. 获得：https://项目名.pages.dev/

4. 员工链接：https://项目名.pages.dev/?mode=submit
